# Octopus Enterprise Web UI 人工功能测试文档

> 版本：v1.0 | 编写日期：2026-03-08 | 编写人：QualityEngineer
>
> 本文档用于指导测试人员/产品经理对 Octopus AI 管理控制台进行全面的人工功能测试。

---

## 测试环境

| 项目 | 说明 |
|------|------|
| **访问地址** | http://localhost:3001 |
| **后端 API** | http://localhost:18790 |
| **管理员账号** | 用户名：`admin`，密码：`password123` |
| **普通用户账号** | 用户名：`baizh`，密码：`password123`（角色为 user，非 admin） |
| **浏览器要求** | Chrome 100+ / Firefox 100+ / Edge 100+，建议分辨率 1920x1080 |
| **启动命令** | 项目根目录执行 `./start-dev.sh start` |
| **停止命令** | `./start-dev.sh stop` |

### 测试前准备

1. 确认后端服务已启动：`./start-dev.sh status` 应显示所有服务运行中
2. 确认健康检查通过：`curl http://localhost:18790/health` 应返回 `{"status":"ok"}`
3. 打开浏览器访问 http://localhost:3001，应看到登录页面
4. 准备测试文件：一个 .txt 文件、一个 .csv 文件、一个 .png 图片（均小于 20MB）

## 优先级说明

| 优先级 | 含义 | 说明 |
|--------|------|------|
| **P0** | 阻塞级 | 影响系统基本可用性，必须通过 |
| **P1** | 核心功能 | 核心业务流程，必须通过 |
| **P2** | 重要功能 | 重要但非核心，应尽量通过 |
| **P3** | 一般功能 | 辅助功能、边界场景 |

## 一、登录页面

### UI-LOGIN-001 正常登录（管理员）
- **优先级**: P0
- **操作步骤**:
  1. 打开浏览器访问 http://localhost:3001
  2. 确认页面显示：左侧紫色渐变品牌区（含 Octopus AI Logo、4 个特性卡片）、右侧白色登录表单区
  3. 在用户名输入框输入 `admin`
  4. 在密码输入框输入 `password123`
  5. 点击「登 录」按钮
- **预期结果**:
  - 按钮显示加载动画和「登录中...」文字
  - 登录成功后跳转到聊天主页面（/chat）
  - 页面左侧显示会话列表侧边栏
- **注意事项**: 首次登录可能需要 2-3 秒等待后端初始化

### UI-LOGIN-002 正常登录（普通用户）
- **优先级**: P0
- **操作步骤**:
  1. 访问 http://localhost:3001
  2. 输入用户名 `baizh`，密码 `password123`
  3. 点击「登 录」
- **预期结果**: 登录成功，跳转到聊天页面
- **注意事项**: baizh 的密码是 `password123`，不是 `baizh`

### UI-LOGIN-003 错误密码登录
- **优先级**: P0
- **操作步骤**:
  1. 访问登录页面
  2. 输入用户名 `admin`，密码 `wrongpassword`
  3. 点击「登 录」
- **预期结果**:
  - 登录按钮恢复正常状态
  - 表单上方出现红色错误提示框，显示错误信息（如「用户名或密码错误」）
  - 页面不跳转，保持在登录页

### UI-LOGIN-004 空输入提交
- **优先级**: P1
- **操作步骤**:
  1. 访问登录页面
  2. 不输入任何内容，直接点击「登 录」
- **预期结果**: 显示错误提示「请输入用户名和密码」
- **注意事项**: 仅输入用户名或仅输入密码也应显示该提示

### UI-LOGIN-005 Enter 键提交
- **优先级**: P2
- **操作步骤**:
  1. 在用户名输入框输入 `admin`
  2. Tab 切换到密码输入框，输入 `password123`
  3. 按 Enter 键
- **预期结果**: 触发登录，与点击按钮效果相同

### UI-LOGIN-006 登录页品牌展示区
- **优先级**: P3
- **操作步骤**:
  1. 使用大屏幕（宽度 > 1024px）访问登录页
  2. 观察左侧品牌展示区
- **预期结果**:
  - 显示紫色渐变背景
  - Logo 区域有缩放动画
  - 4 个特性卡片（智能对话、工具集成、数据分析、企业安全）悬停时有上浮效果
  - 底部显示「Powered by DeepSeek & Claude」
  - 有浮动圆形装饰动画和光带扫过效果

### UI-LOGIN-007 移动端登录页适配
- **优先级**: P3
- **操作步骤**:
  1. 将浏览器窗口缩小到 < 1024px 宽度
  2. 观察登录页面
- **预期结果**: 左侧品牌展示区隐藏，表单区域居中显示，顶部出现 Octopus AI 小 Logo

## 二、聊天页面（/chat）

### 2.1 左侧侧边栏

#### UI-CHAT-001 Agent 选择器
- **优先级**: P1
- **操作步骤**:
  1. 登录后进入聊天页面
  2. 查看侧边栏顶部的 Agent 下拉选择器
  3. 点击下拉选择器，切换到其他 Agent
- **预期结果**:
  - 下拉列表显示所有可用 Agent，每个 Agent 显示 Emoji + 名称
  - 默认 Agent 旁边显示「默认」标签
  - 切换 Agent 后，当前会话和消息被清除
  - 会话列表更新为新 Agent 的会话

#### UI-CHAT-002 新建对话
- **优先级**: P0
- **操作步骤**:
  1. 点击侧边栏中的「新对话」按钮（带 + 图标）
- **预期结果**:
  - 右侧消息区显示欢迎屏
  - 消息列表清空
  - 搜索面板（如果展开的话）关闭

#### UI-CHAT-003 搜索历史消息
- **优先级**: P2
- **操作步骤**:
  1. 先发送几条消息，确保有历史记录
  2. 点击「新对话」按钮右侧的搜索图标按钮
  3. 在展开的搜索框中输入关键词
  4. 按 Enter 或点击搜索按钮
- **预期结果**:
  - 搜索面板展开，显示搜索输入框和搜索按钮
  - 搜索结果显示在下方列表中，每条结果显示会话 ID 和消息片段
  - 点击搜索结果可跳转到对应会话

#### UI-CHAT-004 会话列表切换
- **优先级**: P1
- **操作步骤**:
  1. 确保有至少 2 个会话
  2. 点击不同的会话条目
- **预期结果**:
  - 被选中的会话高亮显示（背景色加深）
  - 右侧消息区加载该会话的历史消息
  - 每个会话条目显示：消息图标、标题、消息条数、最后活跃时间

#### UI-CHAT-005 会话重命名
- **优先级**: P2
- **操作步骤**:
  1. 将鼠标悬停在某个会话条目上
  2. 点击右侧出现的「...」操作按钮
  3. 在下拉菜单中选择「重命名」
  4. 在变为输入框后输入新标题
  5. 按 Enter 确认
- **预期结果**:
  - 点击「重命名」后，标题变为可编辑输入框
  - Enter 确认后标题更新
  - 按 Esc 可取消重命名
  - 失去焦点（点击其他地方）也会提交重命名

#### UI-CHAT-006 AI 生成标题
- **优先级**: P2
- **操作步骤**:
  1. 对一个有消息内容的会话，点击「...」按钮
  2. 选择「AI 生成标题」
- **预期结果**:
  - 显示 toast 提示「标题已生成」
  - 会话标题被 AI 生成的标题替换
  - 会话列表自动刷新

#### UI-CHAT-007 导出会话（Markdown）
- **优先级**: P2
- **操作步骤**:
  1. 对一个有消息的会话，点击「...」按钮
  2. 选择「导出 Markdown」
- **预期结果**:
  - 浏览器自动下载一个 .md 文件
  - 显示 toast 提示「导出成功」

#### UI-CHAT-008 导出会话（JSON）
- **优先级**: P3
- **操作步骤**:
  1. 对一个有消息的会话，点击「...」按钮
  2. 选择「导出 JSON」
- **预期结果**:
  - 浏览器自动下载一个 .json 文件
  - 显示 toast 提示「导出成功」

#### UI-CHAT-009 删除会话
- **优先级**: P1
- **操作步骤**:
  1. 对一个会话，点击「...」按钮
  2. 选择红色的「删除」选项
  3. 在弹出的确认对话框中点击「确定」
- **预期结果**:
  - 弹出浏览器原生确认框「确定删除此会话？」
  - 确认后会话从列表中移除
  - 如果删除的是当前选中的会话，消息区清空
  - 显示 toast 提示「会话已删除」

#### UI-CHAT-010 折叠/展开侧边栏
- **优先级**: P2
- **操作步骤**:
  1. 点击侧边栏右侧边缘的折叠按钮（竖线小按钮）
  2. 再次点击展开按钮
- **预期结果**:
  - 点击后侧边栏收起，消息区占据全宽
  - 折叠按钮变为展开图标
  - 再次点击后侧边栏恢复显示
  - 过渡动画平滑

### 2.2 欢迎屏

#### UI-CHAT-011 欢迎页面展示
- **优先级**: P1
- **操作步骤**:
  1. 新建对话（或首次登录时无会话）
  2. 观察右侧消息区
- **预期结果**:
  - 显示 Octopus AI Logo（带动画）
  - 显示「你好，{用户名}」问候语
  - 显示「我是 {Agent名称}，可以帮助你完成各种任务」
  - 显示 4 个建议提示卡片（2x2 网格）

#### UI-CHAT-012 点击建议提示
- **优先级**: P2
- **操作步骤**:
  1. 在欢迎页面，点击其中一个建议提示（如「帮我分析今日的电力负荷数据趋势」）
- **预期结果**:
  - 直接发送该提示文本作为消息
  - 欢迎页切换为消息列表
  - AI 开始流式回复

### 2.3 消息收发

#### UI-CHAT-013 发送文本消息
- **优先级**: P0
- **操作步骤**:
  1. 在输入框中输入一段文字（如「你好，请介绍你自己」）
  2. 点击发送按钮（纸飞机图标）
- **预期结果**:
  - 用户消息显示在右侧，蓝色气泡，圆角
  - AI 回复显示在左侧，灰色气泡，带 Octopus Logo 头像
  - AI 回复过程中显示三个跳动点加载动画
  - 流式输出文字逐步显示
  - 消息区自动滚动到底部

#### UI-CHAT-014 Enter 发送 / Shift+Enter 换行
- **优先级**: P1
- **操作步骤**:
  1. 在输入框输入第一行文字
  2. 按 Shift+Enter 换行
  3. 输入第二行文字
  4. 按 Enter 发送
- **预期结果**:
  - Shift+Enter 插入换行，不发送
  - Enter 发送消息
  - 发送的消息保留换行格式

#### UI-CHAT-015 发送按钮禁用状态
- **优先级**: P2
- **操作步骤**:
  1. 清空输入框，且无附件
  2. 观察发送按钮状态
  3. 在 AI 正在回复时观察输入框和发送按钮
- **预期结果**:
  - 输入框为空且无附件时，发送按钮为禁用状态
  - AI 正在回复时（流式输出中），输入框和附件按钮为禁用状态

#### UI-CHAT-016 输入框自动高度调整
- **优先级**: P3
- **操作步骤**:
  1. 在输入框中输入多行文字（超过 3 行）
- **预期结果**:
  - 输入框高度自动增加，最大不超过约 200px
  - 发送后输入框恢复初始高度

#### UI-CHAT-017 思考过程显示
- **优先级**: P1
- **操作步骤**:
  1. 如果当前 Agent 使用 DeepSeek R1 模型（支持思考模式）
  2. 发送一个需要推理的问题（如「请计算 1+1=?，并解释为什么」）
- **预期结果**:
  - AI 回复气泡中先显示「正在思考...」和大脑图标
  - 思考过程以较小字体、灰色缩进显示
  - 思考结束后最终回复正常显示
  - 点击「思考过程」可折叠/展开
- **注意事项**: 仅 DeepSeek R1 模型会返回思考过程

### 2.4 附件功能

#### UI-CHAT-018 添加附件
- **优先级**: P1
- **操作步骤**:
  1. 点击输入框左侧的回形针图标按钮
  2. 在文件选择器中选择一个文本文件（如 .txt）
- **预期结果**:
  - 输入框上方出现附件预览区
  - 显示文件名、文件大小
  - 文件名前有回形针图标

#### UI-CHAT-019 删除单个附件
- **优先级**: P2
- **操作步骤**:
  1. 添加一个附件后
  2. 点击附件标签上的 X 按钮
- **预期结果**: 该附件从预览区移除

#### UI-CHAT-020 带附件发送消息
- **优先级**: P1
- **操作步骤**:
  1. 添加一个 .txt 附件
  2. 在输入框输入「请分析这个文件的内容」
  3. 发送消息
- **预期结果**:
  - 用户消息中显示附件标记（如 `[附件] test.txt`）
  - AI 能基于附件内容进行回复
  - 发送后附件预览区清空

#### UI-CHAT-021 多文件附件
- **优先级**: P3
- **操作步骤**:
  1. 点击附件按钮，选择多个文件（文件选择器支持多选）
  2. 发送
- **预期结果**: 所有文件都作为附件附加，消息中显示所有附件名

### 2.5 斜杠命令

#### UI-CHAT-022 斜杠命令菜单显示
- **优先级**: P1
- **操作步骤**:
  1. 在输入框第一个字符位置输入 `/`
- **预期结果**:
  - 输入框上方弹出命令菜单
  - 显示两个顶级命令：`/mcp`（可用 MCP 数量）和 `/skill`（可用 Skill 数量）
  - 每个命令右侧有展开箭头，表示有子项

#### UI-CHAT-023 斜杠命令 - /mcp 子项
- **优先级**: P1
- **操作步骤**:
  1. 输入 `/mcp ` （注意带空格）
  2. 或先输入 `/`，用方向键选中 /mcp，按 Tab/Enter
- **预期结果**:
  - 菜单显示当前 Agent 可用的 MCP 服务器列表
  - 每项显示名称和描述
  - 输入字符可过滤列表

#### UI-CHAT-024 斜杠命令 - /skill 子项
- **优先级**: P1
- **操作步骤**:
  1. 输入 `/skill `
- **预期结果**: 菜单显示当前 Agent 可用的技能列表

#### UI-CHAT-025 斜杠命令键盘导航
- **优先级**: P2
- **操作步骤**:
  1. 输入 `/` 显示菜单
  2. 按上下方向键导航
  3. 按 Tab 或 Enter 选中
  4. 按 Esc 关闭菜单
- **预期结果**:
  - 上下方向键移动高亮项
  - Tab/Enter 选中当前高亮项，若为组命令则展开子项（填入 `/mcp ` 或 `/skill `），否则填入完整命令
  - Esc 关闭菜单但保留已输入内容

#### UI-CHAT-026 斜杠命令选择后
- **优先级**: P2
- **操作步骤**:
  1. 通过斜杠命令选择一个 MCP 服务器（如 `/mcp 高德地图`）
  2. 在命令后追加文字，发送
- **预期结果**:
  - 选中后命令填入输入框并追加空格，光标在末尾
  - 可继续输入内容
  - 发送后 AI 使用指定的 MCP/Skill 处理

### 2.6 提醒系统

#### UI-CHAT-027 提醒浮层
- **优先级**: P2
- **操作步骤**:
  1. 确保有已到期的提醒（通过定时任务或手动创建）
  2. 等待 30 秒轮询周期
- **预期结果**:
  - 右下角出现浮层卡片，显示铃铛图标、提醒标题和内容
  - 可能弹出浏览器原生通知（需要允许通知权限）
  - 点击浮层上的 X 按钮可关闭
- **注意事项**: 提醒每 30 秒轮询一次，可能有延迟

### 2.7 AI 回复响应

#### UI-CHAT-028 AI 回复等待时间
- **优先级**: P1
- **操作步骤**:
  1. 发送一条普通问题
  2. 观察响应时间
- **预期结果**:
  - AI 回复通常在 10-30 秒内开始流式输出
  - 等待期间显示跳动点动画
  - 输入框显示为禁用状态
- **注意事项**: DeepSeek 模型回复可能需要 10-30 秒

#### UI-CHAT-029 新会话自动生成标题
- **优先级**: P2
- **操作步骤**:
  1. 点击「新对话」创建新会话
  2. 发送第一条消息
  3. 等待 AI 回复完成
- **预期结果**:
  - 约 800ms 后侧边栏中的会话标题自动更新为 AI 生成的标题
  - 不再显示默认的会话 ID

#### UI-SET-004 管理员菜单对普通用户不可见
- **优先级**: P0

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 登录后页面空白 | 检查后端服务是否启动：`./start-dev.sh status` |
| AI 回复超时（超过 60 秒） | DeepSeek 模型在高负载时可能变慢，刷新重试 |
| 会话标题显示为 ID | 等待自动生成标题（约 800ms），或手动点击「AI 生成标题」 |
| 斜杠命令菜单不显示 | 确保 `/` 是输入框中的第一个字符 |
| 上传文件失败 | 检查文件大小是否超过 20MB |
| 管理员菜单看不到 | 确认使用 admin 账号登录（baizh 是普通用户） |
| 提醒不触发 | 提醒每 30 秒轮询一次，可能有延迟 |
| 心跳巡检创建失败 | 每个 Agent 最多一个心跳任务 |
| 数据库连接创建失败 | 检查必填字段，新建时密码为必填 |
| Docker sandbox 相关错误 | Agent 执行命令在 Docker 容器中，确认 Docker 服务正常 |
