# 标书生成 Skill 设计文档

**日期**: 2026-03-13
**状态**: 已批准

---

## 概述

基于已有标书模板（.docx），结合新项目招标需求，直接在原文档副本上修改生成新标书。保留原文档所有格式、表格、样式，仅替换/更新文本内容。

## 设计目标

- 输入：历史标书模板（.docx） + 新项目招标说明（对话或上传文件）
- 输出：新标书（.docx），保留原始格式
- 支持段落文本和表格内容的替换
- 可选大纲确认步骤

## 架构

### 文件结构

## 组件设计

### 1. extract.py — 文档结构提取

**输入参数**:
- `--input`: 源 .docx 文件路径

**输出**: JSON 到 stdout

```json
{
  "paragraphs": [
    {
      "index": 0,
      "text": "XXX公司2025年度设备采购投标文件",
      "style": "Heading 1"
    },
    {
      "index": 1,
      "text": "一、投标人基本情况",
      "style": "Heading 2"
    }
  ],
  "tables": [
    {
      "index": 0,
      "location_after_paragraph": 5,
      "rows": [
        ["公司名称", "XXX有限公司"],
        ["成立日期", "2010年1月"]
      ]
    }
  ]
}
```

**实现要点**:
- 遍历 `document.paragraphs`，记录 index、text、style name
- 遍历 `document.tables`，记录每个 table 的行列文本
- 通过 `document.element.body` 的子元素顺序确定 table 在段落间的位置

### 2. render.py — 文档修改渲染

**输入参数**:
- `--input`: 源 .docx 文件路径
- `--output`: 输出 .docx 文件路径
- `--instructions`: 修改指令 JSON 文件路径

**修改指令格式**:

```json
{
  "paragraphs": [
    {
      "index": 0,
      "action": "replace",
      "new_text": "XXX公司2026年度设备采购投标文件"
    },
    {
      "index": 15,
      "action": "replace",
      "new_text": "项目预算：人民币叁佰万元整"
    }
  ],
  "tables": [
    {
      "table_index": 0,
      "cell_updates": [
        {
          "row": 0,
          "col": 1,
          "new_text": "YYY有限公司"
        }
      ]
    }
  ]
}
```

**实现要点**:
- 复制源文件到输出路径（`shutil.copy2`）
- 打开副本进行修改
- 段落替换：保留第一个 run 的格式，清空后续 runs，将新文本写入第一个 run
- 表格替换：定位 `table[i].rows[r].cells[c]`，同样保留首个 run 格式
- 如果段落/单元格无 run，创建新 run 继承段落样式

# 标书生成助手

> 路径约定：`{{SKILL_DIR}}` 代表本技能根目录。

## 功能概述

基于用户提供的历史标书模板（.docx）和新项目招标需求，直接在原文档副本上修改生成新标书。

核心流程：
- **结构提取** — 解析标书模板的段落和表格结构
- **需求分析** — AI 对比新旧需求，确定修改点
- **大纲确认** — 展示修改大纲供用户审核（可选）
- **渲染生成** — 在文档副本上执行修改，输出新 .docx

## When NOT to Use This Skill

- 没有历史标书模板，需要从零撰写 → 直接用对话生成文本
- 需要修改 PDF 格式标书 → 先转换为 .docx
- 仅需简单文字替换（如替换公司名） → 直接用文本编辑

## 工具

### extract — 提取文档结构

提取 .docx 文件的段落和表格结构，返回 JSON。

**调用方式**：
```
run_skill(skill_name="bid-document", args="extract --input <源文档路径>")
```

**参数**：
| 参数 | 必填 | 说明 |
|------|------|------|
| `--input` | 是 | 源 .docx 文件路径 |

**输出**：JSON 格式的文档结构（段落列表 + 表格列表）

### render — 生成新标书

在源文档副本上执行修改指令，生成新文档。

**调用方式**：
```
run_skill(skill_name="bid-document", args="render --input <源文档> --output <输出路径> --instructions <指令JSON>")
```

**参数**：
| 参数 | 必填 | 说明 |
|------|------|------|
| `--input` | 是 | 源 .docx 文件路径 |
| `--output` | 是 | 输出 .docx 文件路径 |
| `--instructions` | 是 | 修改指令 JSON 文件路径 |

## 修改指令格式

```json
{
  "paragraphs": [
    {"index": 0, "action": "replace", "new_text": "新标题文本"}
  ],
  "tables": [
    {
      "table_index": 0,
      "cell_updates": [
        {"row": 0, "col": 1, "new_text": "新单元格内容"}
      ]
    }
  ]
}
```

**注意**：
- `index` 和 `table_index` 必须对应 extract 输出的 `index` 字段
- `location_after_paragraph` 仅供 AI 理解上下文，不用于 render 定位
- 合并单元格区域内多个坐标指向同一单元格，后写覆盖先写

## 格式保留策略

| 元素 | 策略 |
|------|------|
| 段落文字 | 保留首个 run 的字体、字号、加粗、颜色等属性 |
| 表格文字 | 同段落策略，保留单元格首个 run 格式 |
| 表格结构 | 不增删行列，仅修改单元格文本 |
| 页眉页脚 | 不修改，原样保留 |
| 图片 | 不修改，原样保留 |
| 页面布局 | 不修改，原样保留 |

## 常见问题

### Q: 段落内有部分加粗/斜体，替换后会保留吗？
A: 仅保留首个 run 的格式。如原文 "公司名称：**XXX公司**"，替换后整段使用 "公司名称" 部分的格式。

### Q: 表格中的合并单元格怎么处理？
A: python-docx 中合并区域内的所有坐标指向同一单元格。避免对合并区域内多个坐标发送不同 new_text。

### Q: 文档太大，extract 输出被截断怎么办？
A: SkillExecutor 有 1MB stdout 限制。大型标书建议分章节提取，或仅关注需要修改的部分。

## 错误处理

两个脚本遵循统一的错误约定：

| 情况 | 退出码 | 输出 |
|------|--------|------|
| 成功 | 0 | stdout: JSON（extract）或成功消息（render） |
| 输入文件不存在/无效 | 1 | stderr: 错误描述 |
| 指令 JSON 格式错误 | 1 | stderr: 错误描述 |
| python-docx 解析失败 | 1 | stderr: 错误描述 |

SkillExecutor 通过 `exitCode === 0` 判断成功，stderr 内容会返回给 AI agent。

## 限制与未来扩展

**当前限制**:
- 仅支持文本替换，不支持增删表格行列
- 不处理图片替换
- 不处理页眉页脚修改
- 段落替换保留首个 run 格式，段落内混合格式（如部分加粗）会丢失
- 合并单元格：写入合并区域内的多个坐标实际指向同一单元格，后写覆盖先写
- SkillExecutor stdout 限制 1MB，超大文档的 extract 输出可能被截断

**未来可扩展**:
- 表格行列增删（如投标清单项目数量变化）
- 图片替换（如公章、资质证书）
- 多文档合并生成
